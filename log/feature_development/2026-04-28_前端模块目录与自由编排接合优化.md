# 前端模块目录与自由编排接合优化

## 目标

根据 `GUIDELINES.md`、`WORKFLOW_DESIGN.md`、`docs/workflow/STAGE_PROTOCOL.md` 与 `docs/agent_skills/historyresearch-workspace/`，把前端从少量固定功能入口扩展为可自由配置的历史研究工作台。

## 修改范围

- `frontend/src/data/moduleCatalog.ts`
- `frontend/src/pages/ManualWorkspace/`
- `frontend/src/pages/AgentSolo/`
- `frontend/src/types/models.ts`

## 关键结果

- 新增 workspace module catalog，覆盖三类能力：
  - TaskManager 可直接执行任务：NER、academic note、paper polish、citation normalize、historical citation、OCR correction、summary、reverse outline、style transfer、virtual persona、entity disambiguation。
  - 七阶段 workflow 节点：collect、organize、extract、examine、write、polish、format。
  - package-only 模块节点：PDF image conversion、Unified OCR、NDL OCR batch、layout analyzer、date matcher、classical OCR training、biography extraction、historical speech、embedding、citation network、citation formatter、Obsidian、field explorer、environment checker、artifact manager。
- 手动挡新增类似 Dify 的自由工作流编排界面：
  - 可从模块目录添加节点。
  - 可调整节点顺序。
  - 可配置 input binding、output binding、backend、provider、review gate。
  - 自动生成顺序边和 workflow blueprint JSON。
  - 可将自定义流程加入任务中心。
- agent solo 模式新增 workspace skill 配置器：
  - 可选择 agent 能调用的模块范围。
  - 可配置 allowed backends。
  - 可配置外部检索、下载、写入 vault、付费 API、artifact 写入等权限。
  - 自动生成 `SKILL.md` 草案和配置 JSON。
  - 可将 skill 配置复核任务加入任务中心。

## 隐私与合规

- 未读取或暴露 `secrets/`。
- 前端只记录模块能力、脱敏路径、backend/provider 名称和权限布尔值。
- 外部检索、下载、写入 vault、付费 API 等动作均在 UI 中作为显式权限或 review gate 暴露。
- package-only 模块目前以前端蓝图方式接合，后续应由后端 job/package endpoint 承接真实执行。

## 验证

- `npm run lint`
- `npm run test -- --run`
- `npm run build`

以上均已通过。Vite 仍提示 Ant Design chunk 偏大，属于前端体积优化后续项，不影响当前功能。

## 后续建议

- 后端新增 `/api/jobs` 与 `/api/workflow/blueprints`，让当前前端 blueprint 真正落地执行和恢复。
- 为 package-only 模块逐步补齐统一 task registry 映射，减少前端对静态目录的依赖。
- 引入 React Flow 或同类库后，可将当前顺序节点编辑升级为真正的可视化连线画布。
