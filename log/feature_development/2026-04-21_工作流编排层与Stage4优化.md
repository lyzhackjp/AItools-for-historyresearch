# 2026-04-21 工作流编排层与 Stage4 优化

## 背景

在 Stage 3 已经接入统一任务层之后，workflow 主线仍有两个明显短板：

1. `WorkflowOrchestrator` 和具体阶段对象都在改写阶段状态，生命周期职责重叠
2. `Stage4Examine` 运行后没有把执行摘要、review 项和质量标记稳定写回项目状态

这会影响：

- checkpoint 恢复
- artifact 追踪
- 前端展示
- 后续阶段对 citation/outline 结果的复用

## 本轮目标

1. 把新的 workflow 方向先写回 `MODULE_OPTIMIZATION_DESIGN_2026-04-21.md`
2. 重构 `WorkflowOrchestrator`，让其统一负责 checkpoint 和 artifact 登记
3. 重构 `Stage4Examine`，让其输出结构化执行摘要与 review 信号
4. 增加正式测试覆盖 orchestrator 与 Stage 4 的新行为

## 具体改动

### 1. 锚点文档补充

在 `docs/project/MODULE_OPTIMIZATION_DESIGN_2026-04-21.md` 中新增了：

- “编排层与阶段层的生命周期职责仍然重叠”
- `workflow_orchestrator.py` 的第二条新增约束
- `stage4_examine.py` 的新增约束
- Phase 3 新的交付标准：orchestrator 需统一登记 checkpoint、导出产物和异常摘要

### 2. WorkflowOrchestrator 重构

重写 `tools/workflow/workflow_orchestrator.py`，核心变化如下：

- `run_stage()` 不再抢占业务执行逻辑，而是专注：
  - 记录 orchestrator 调用时间
  - 统一保存阶段 checkpoint
  - 回写阶段结果摘要
  - 记录异常摘要
- 新增 workflow artifact 登记能力：
  - `workflow_checkpoint`
  - `stage_output`
  - `export`
- 对 Stage 5 / Stage 7 的正式输出文件进行统一 artifact 挂载
- 对没有显式 `mark_stage_done()` 的成功阶段补做兜底状态修正

### 3. Stage4Examine 重构

重写 `tools/workflow/stages/stage4_examine.py`，核心变化如下：

- 输出 `capability_snapshot`
- 输出 `execution_summary`
  - citation network 节点数
  - 边数
  - 核心文献数
  - 孤立文献数
  - outline review 的 logical gaps / deviation flags
- 将逻辑缺口和偏离项写入 `project.review_queue`
- 将关键风险写入 `project.quality_flags`
- 将 warnings 写入 `stage_metadata`

### 4. 测试补充

新增 `tests/test_workflow_orchestrator_stage4.py`，验证：

- orchestrator 是否记录 checkpoint path
- orchestrator 是否注册 workflow checkpoint artifact
- stage4 是否写入 execution summary
- stage4 是否将 outline review 风险写入 review queue 和 quality flags
