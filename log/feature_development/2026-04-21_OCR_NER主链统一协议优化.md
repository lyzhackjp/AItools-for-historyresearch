## 本轮新增设计方向

已先更新设计锚点文档，新增以下约束：

1. `unified_ocr_processor.py` 不再只是 NDL 双模型切换器，而要成为统一 OCR 能力发现与结果协议中心。
2. `ner_processor.py` 不再停留在 `test_mode + provider` 二分法，而要成为统一多后端 NER 门面。

4. `app/app.py` 禁止在导入阶段直接实例化 OCR、LLM、PDF 等重模块。

## 本轮代码变更

### 1. `modules/unified_ocr_processor.py`

- 重构为统一 OCR 门面层
- 增加统一引擎注册表
- 统一声明以下 OCR 引擎:
  - `tesseract`
  - `llm_ocr`
  - `ndlocr_lite`
  - `ndlkotenocr_lite`
- 增加统一能力发现接口:
  - `get_available_models()`
  - `get_capabilities()`
  - `get_model_info()`
- 统一结果结构:
  - `backend_kind`
  - `provider`
  - `needs_review`
  - `metadata`
  - `attempted_models`

### 2. `modules/ner_processor.py`

- 重构为统一多后端 NER 门面
- 接入 `UnifiedTaskExecutor`
- 保留既有公开方法，如:
  - `recognize_historical_entities`
  - `classify_entities`
  - `extract_entity_relationships`
  - `batch_process_documents`
- 将规则识别、词典补强和统一执行结果合并为同一输出结构
- 为实体统一补充:
  - `entity`
  - `text`
  - `category`
  - `confidence`
  - `backend`
  - `provider`
  - `model`
  - `needs_review`

### 3. `modules/ner_disambiguation.py`

- 重写为兼容双接口版本
- 保留 `EntityDisambiguator`
- 新增 `NERDisambiguation` 兼容封装，满足 `Stage3Extract` 当前调用格式
- 统一返回:
  - `original_entity`
  - `disambiguated_type`
  - `standard_name`
  - `confidence`

### 4. `app/app.py`

- 将以下重模块改为 lazy service:
  - `DocProcessor`
  - `LLMClient`
  - `PDFProcessor`
  - `OCRProcessor`
  - `DataStructurer`
  - `UnifiedOCRProcessor`
  - `NDLDownloadModule`
- 避免 API 导入阶段直接触发 OCR 和模型初始化

### 5. 测试

- 新增 `tests/test_ocr_ner_integration.py`
- 验证:
  - OCR 多引擎能力发现
  - NER 多后端能力发现
  - NER 标准化输出
  - `NERDisambiguation` 与工作流兼容
