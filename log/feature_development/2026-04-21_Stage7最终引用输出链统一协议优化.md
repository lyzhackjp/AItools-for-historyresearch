# 2026-04-21 Stage7 最终引用输出链统一协议优化

## 背景

在前几轮已经完成：

- 统一任务执行层
- OCR -> NER 主链
- Stage 2 / 3 / 4 workflow 元数据回写
- citation schema 主线
- paper polish / reverse outline / Stage 5 / Stage 6 收口

之后，最终引用输出链仍存在一个明显缺口：`stage7_format.py` 仍保留着局部 reference string 拼接逻辑，没有真正消费统一 citation record，也没有把最终输出链的 review 信号、artifact、execution summary 写回 workflow。

这会影响：

- 写作链最后一段与前面 citation schema 主线的衔接
- Stage 7 的可追踪性、可恢复性与前端可展示性
- 最终输出产物的统一审计与质量标记

## 本轮目标

1. 先把 Stage 7 的新增约束补回锚点文档
2. 重写 `stage7_format.py`，让它正式消费统一 citation record
3. 补齐 Stage 7 的 metadata / review_queue / quality_flags / artifact 写回
4. 增加正式测试验证 Stage 7 的新链路

## 具体改动

### 1. 锚点文档补充

在 `docs/project/MODULE_OPTIMIZATION_DESIGN_2026-04-21.md` 中补记了 Stage 7 新约束，明确：

- Stage 7 必须正式消费统一 citation record
- 需要写回：
  - `normalized_citation_records`
  - `formatted_reference_count`
  - `records_needing_review`
  - `output_artifacts`
  - `execution_summary`
- 最终输入文本必须遵循：
  - `style_transferred_draft -> polished_draft -> paper_draft`

### 2. Stage7Format 重写

重写 `tools/workflow/stages/stage7_format.py`，当前实现的核心行为：

1. 统一选择最终输入稿：
   - 优先 `style_transferred_draft`
   - 其次 `polished_draft`
   - 最后 `paper_draft`
2. 将 `project.literature` 转换为统一 citation record
3. 基于统一 citation record 使用 `CitationFormatter.format_record(...)` 生成 references
4. 如原稿末尾已有 references 章节，则先剥离，再统一追加新的 references
5. 尝试导出 Word 文件并登记 artifact
6. 将 execution summary、normalized record 摘要、warnings、artifacts 写回 `stage_metadata`

### 3. Stage 7 回写内容

当前 Stage 7 已经会回写：

- `capability_snapshot`
- `requested_execution`
- `normalized_citation_records`
- `execution_summary`
- `warnings`
- `output_artifacts`

并在必要时写入：

- `review_queue`
- `quality_flags`
- `artifacts`

### 4. 评审与质量信号

当 citation record 缺关键字段或置信度较低时，Stage 7 会：

- 在 `review_queue` 中加入 citation review 项
- 在 `quality_flags` 中加入 `stage7_citation_review_needed`

当 literature 为空时，也会标记：

- `stage7_no_references`

### 5. 测试补充

新增正式测试：

- `tests/test_stage7_format_chain.py`

该测试验证：

- Stage 7 是否优先使用 `style_transferred_draft`
- 是否附加标准 references
- 是否写回 `normalized_citation_records`
- 是否写回 `execution_summary`
- 是否成功完成 Stage 7 状态迁移
