# 2026-04-21 笔记链统一任务层优化

## 背景

在 workflow 的 Stage 2 / Stage 3 / Stage 4 主线逐步协议化之后，笔记链仍有三个明显短板：

1. `academic_note_generator.py` 仍保留旧式直连 LLM 和环境变量路径
2. `obsidian_integration.py` 同时承担文件系统、模板、内容处理等过多职责
3. `stage2_organize.py` 对笔记生成器返回值假设不稳定，且缺少阶段级摘要、review 标记和 artifact 回收

这些问题会影响：

- 笔记链的多后端接入
- Stage 2 与后续 workflow 阶段的数据衔接
- vault 导出追踪与人工复核

## 本轮目标

1. 先把笔记链新增方向写回 `MODULE_OPTIMIZATION_DESIGN_2026-04-21.md`
2. 重构 `AcademicNoteGenerator`，将其变成统一任务层门面
3. 重构 `ObsidianIntegration`，收口为稳定 vault 集成层
4. 重构 `Stage2Organize`，让其统一消费结构化笔记结果并回写 workflow 元数据
5. 增加正式测试覆盖笔记链的新行为

## 具体改动

### 1. 锚点文档补充

在 `docs/project/MODULE_OPTIMIZATION_DESIGN_2026-04-21.md` 中新增了：

- `academic_note_generator.py` 的新增约束
- `obsidian_integration.py` 的新增约束
- `stage2_organize.py` 的新增约束
- Phase 3 新的交付标准：学术笔记链需统一输出 `markdown/entities/export_summary/backend/provider/model`

### 2. AcademicNoteGenerator 重构

重写 `modules/academic_note_generator.py`，保留原有公共方法名，同时改为：

- 通过 `TaskManager` 执行 `academic_note`
- 通过 `TaskManager` 执行 `ner`
- 统一返回：
  - `markdown`
  - `entities`
  - `keywords`
  - `backend`
  - `provider`
  - `model`
  - `needs_review`

兼容保留的方法包括：

- `generate_reading_note`
- `extract_entities`
- `build_knowledge_graph`
- `apply_note_template`
- `create_bidirectional_links`
- `batch_process`
- `create_academic_note_generator`

### 3. ObsidianIntegration 重构

重写 `modules/obsidian_integration.py`，将其收口为 vault 文件系统集成层，专注：

- vault 初始化
- safe note write
- 搜索与 backlinks
- graph scan
- markdown 批量导入
- JSON 导出

并修复了 vault 基础结构初始化问题，保证 `.obsidian/` 和常用目录能稳定创建。

### 4. Stage2Organize 重构

重写 `tools/workflow/stages/stage2_organize.py`，核心变化如下：

- 统一消费结构化 note 结果，而不是假设返回纯字符串
- 将以下信息写回 `project.stage_metadata["stage2"]`：
  - `capability_snapshot`
  - `execution_summary.notes`
  - `execution_summary.vault_export`
  - `execution_summary.citations`
  - `warnings`
- 将需要人工复核的 note 写入 `review_queue`
- 将笔记链风险写入 `quality_flags`
- 将 vault 路径注册到 `artifacts`

### 5. 测试补充

新增 `tests/test_stage2_note_chain.py`，验证：

- `AcademicNoteGenerator` 是否正确规范化统一任务层输出
- `Stage2Organize` 是否写入 note summary
- 是否写入 vault export summary
- 是否注册 Obsidian vault artifact
