# 隐私、日志与 Artifact 规范

## 不记录的内容

- 真实 API key、token、cookie、账号密码。
- 未脱敏的私密史料全文。
- 可复原的用户敏感材料。
- 外部服务返回中的敏感 header 或凭据。

## 可以记录的内容

- 本地路径。
- 页码、段落号、实体数量、引用数量。
- 后端类型、provider、model。
- 置信度、复核标记、错误类别。
- 脱敏摘要和统计。

## 目录约定

| 目录 | 用途 |
| --- | --- |
| `log/feature_development/` | 优化步骤报告 |
| `output/` | 正式运行产物 |
| `workflow_output/` | 工作流示例产物 |
| `cache/` | 可重建缓存 |
| `temp/`, `tmp/` | 短期临时文件 |
| `archive/`, `archives/` | 已归档材料 |
| `secrets/` | 本地敏感配置，不公开 |

## 清理规则

- 自动化或测试产生的临时脚本，在报告生成后删除或归档。
- 可重建缓存可以删除，不影响项目状态。
- 不确认来源的文件不自动删除。
- 清理行为必须写入报告，至少说明范围、保留项和未清理原因。
