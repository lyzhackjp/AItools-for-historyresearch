# Privacy Rules

- Treat this as a local-first historical research workspace.
- Never open, summarize, quote, copy, or log `secrets/`.
- Do not include real API keys, cookies, account tokens, private source text, or recoverable sensitive material in reports, tests, examples, or logs.
- Logs should record paths, counts, backend metadata, quality flags, errors, and short summaries only.
- Temporary files belong in controlled temp/output locations and must be removed or archived after the formal report is written.
- Prefer dry-run, mocked, or lightweight tests unless the user explicitly authorizes real external calls.
- When using remote APIs, record provider/model/cost-relevant metadata without storing the full private payload.
